# Session: DAB REST Endpoint Path Fix
**Date:** 2026-04-15 @ 15:57  
**Coordinator:** AK  
**Status:** ✅ Resolved

## Problem
App was returning 404 errors on `/api/Disc` endpoint calls. Root cause: Azure Static Web Apps serves DAB REST API at `/data-api/rest/{entity}`, not `/api/{entity}`.

## Changes Applied
1. **app.js**: Updated `API_BASE` from `/api/Disc` → `/data-api/rest/Disc`
2. **staticwebapp.config.json**: 
   - Reverted `/data-api/*` route from 'anonymous' back to 'authenticated'
   - Removed redundant `/api/*` route (was debug-only)
3. **swa-db-connections/staticwebapp.database.config.json**: 
   - Reverted Disc entity permission from 'anonymous' back to 'authenticated'

## Commit
- SHA: cfa3d03
- Message: "fix: use /data-api/rest/Disc path and re-lock auth to authenticated"
- Deployed to main — SWA pipeline triggered

## Key Learning
In Azure Static Web Apps, `/api/*` is reserved for Azure Functions. DAB always uses `/data-api/rest/` for REST endpoints.
