## Decision Context

Tab switcher chosen because the priority approach (photo hides chart) is a UX regression — users lose access to the flight path visualization once they add a photo. Tabs preserve both views with one click, degrade gracefully (tab bar hidden when no photo exists), and follow familiar mobile/web patterns.

CSS additions: ~40 lines (`.image-tab-bar`, `.tab-btn`, `.tab-btn.active`).  
Alpine state additions: `imageTab: 'photo'`, `hasPhoto(disc)` helper, post-upload/remove tab resets.
