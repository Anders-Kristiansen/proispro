## Summary

Danny authored the architectural decision record for user disc photo storage:
- Schema: `user_photo_url TEXT` column on `disc_wear_adjustments` (no new table; 1:1 per bag disc enforced)
- Storage: Supabase Storage bucket `disc-photos` (public), path `{user_id}/{user_disc_id}.{ext}`
- Upload: Direct browser → Supabase Storage via `supabase-js` with `upsert: true`; store returned public URL in DB
- RLS: `storage.objects` policies scoped to own `{user_id}/*` prefix via `storage.foldername(name)[1]`
- Constraints: JPEG/PNG/WEBP, 5MB max, no client-side resize for v1
- Display priority: `wearAdjustment.user_photo_url` → `catalogEntry.pic` → SVG flight chart
- Full ADR committed at `docs/adr-disc-photo-storage.md`

## Decision Context

Path keyed on `user_disc_id` (not `wear_adjustment_id`) to avoid orphaned files if the adjustment row is recreated. Public bucket chosen to eliminate signed-URL expiry complexity for a static app.
