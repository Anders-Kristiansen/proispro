### Recommendations

- **RLS strategy:** Indirect ownership via join tables (collection_discs) avoids denormalization and update anomalies
- **Status as TEXT + CHECK:** Chosen over ENUM for no-downtime constraint updates
- **Non-reactive caching:** Collection disc joins cached separately to keep Alpine state flat and maintainable
- **Supabase-only features:** Collections/Wishlist/ForSale require authentication; no localStorage fallback needed

### Trade-offs Named

- **Complexity vs. Simplicity:** Tab UI adds complexity but keeps single-page paradigm; modal-driven editing chosen over page navigation
- **Flat vs. Nested State:** Cached joins stay flat (simpler state machine) vs. nested structure (more OOP-like but harder to reason about)
- **Schema Flexibility:** TEXT status allows zero-downtime updates vs. ENUM (DDL changes require maintenance windows)
