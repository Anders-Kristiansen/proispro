### Key Implementation Choices

**Sort Format: Combined Strings**
- Used `'speed-desc'` instead of separate `sortBy` + `sortDir` state
- Simpler dropdown UI, easier URL persistence later
- Parsed in `filteredSorted` via `const [field, dir] = this.sortBy.split('-')`

**Grouping: Default Expanded**
- Groups expand by default via `groupExpanded[label] !== false` (undefined = expanded)
- Reduces clicks for typical use case (users want to see all discs)
- `toggleGroup()` flips boolean state

**Tag Colors: Hash Function**
- `tagColor(tag)` hashes tag string to one of 8 OKLCH colors (putter, midrange, fairway, distance, accent, accent2, emerald, iris)
- Applied via inline `:style` binding: `background: oklch(from var(--{color}) l c h / 0.2); color: var(--{color})`
- Uses OKLCH `oklch(from ...)` relative color syntax for semi-transparent backgrounds
- Deterministic: same tag always gets same color

**List View Header: Single Instance**
- Header row shown only for first group via `x-if="groupedDiscs.indexOf(group) === 0"`
- Avoids duplicate headers when grouping is active
- Column headers clickable for sorting (calls `setSortColumn(field)`)

**Advanced Filters: Popover + Click-Outside**
- `@click.outside="showAdvancedPopover = false"` auto-closes
- All filter state lives in top-level Alpine data (not separate component)
- "Clear all filters" button resets all 4 filter variables

**View Mode Switching: Conditional Templates**
- Main container uses `:class="{ 'disc-grid': viewMode === 'grid', ... }"`
- Each view template wrapped in `x-if="viewMode === '...'"` (only one rendered at a time)
- Grid view template preserved from existing code (zero breaking changes)

## Learnings for Future Frontend Implementations

- **Combined sort strings** simplify state management vs. separate field/direction
- **Default expanded groups** reduce cognitive load vs. collapsed defaults
- **Inline dynamic styles** more maintainable than generated CSS classes
- **Single list header** (first group only) prevents visual redundancy when grouping
- **Alpine `x-if` vs. `x-show`:** Used `x-if` for view templates to completely remove DOM (not just hide), reducing memory for large lists
- **Conditional rendering in loops:** Wrapped all 3 view templates in `groupedDiscs` loop; only one rendered at a time via `x-if`
- **Click-outside pattern:** `@click.outside` on popover auto-closes without manual escape handling
- **Autocomplete via datalist:** Simple HTML5 `<datalist>` works well; future: consider custom dropdown for better mobile support
